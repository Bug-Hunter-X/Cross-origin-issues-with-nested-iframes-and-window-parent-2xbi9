This is an example of an uncommon HTML bug that can occur when using nested iframes and cross-origin resources.

Bug: When an iframe loads a resource from a different origin (e.g., a different domain), and that resource tries to access the parent window's properties or methods using window.parent, it may encounter errors due to browser security restrictions (Cross-Origin Resource Sharing or CORS). This is particularly problematic when nested iframes are involved, where the access path is further obscured.

Example:

<iframe src="https://example.com/iframe1.html" id="iframe1"></iframe>
<iframe src="https://another-example.com/iframe2.html" id="iframe2"></iframe>

In this scenario, if iframe2.html attempts to access elements within iframe1.html (or the main page) using window.parent, it might fail, due to the cross-origin restrictions. The specifics of the error message will vary depending on the browser.
